从NexT到Hexo到npm更新了一下

发表于 2016-05-13 Disqus：

NexT更新

最近看到好多同学的Hexo用了新的scheme （还有个叫Muse的scheme），于是也想试一下。查了一下NexT的仓库，版本从0.4.5.2直接跳到了5.0.0。目瞪口呆.jpg。
文档也做的很好看了：NexT文档

更新的方式很简单，直接

因为之前叫hexo-theme-next所以完全不用担心冲突的问题。

但是需要对比修改一下next里的_config.yml，一样的项目可以直接复制，主要需要和发现更新注意的地方有：

  1. 增加的sidebar仅针对Pisces这个scheme，不过为什么不试试这个5.0的新主题呢
  2. 多说的方式变成添加shortname即可，shortname自己在多说后台管理里设置的，可以查到，不再需要添加代码，更加易用
  3. 多说信息如果开启管理员模式需要user_id，这个在dev.duoshuo.com登陆后就可以看到，就是一串数字
  4. 发现之前没注意到的google_site_verification，因为之前是用Google analytics验证的，所以又做了一次验证
  5. Mist下右上角的按钮都多了一个小图标，搜索框也变成了搜索按钮+图标
    等

顺便今天cc98的markdown功能上线，去github又复习了一下markdown语法，发现```的语法可以支持指定语言，正好对之前代码高亮不太满意，于是学习一下。NexT的代码高亮使用的是Tomorrow Theme的主题，而Tomorrow Theme使用的是highlight.js的实现。看了一下highlight.js支持的语言，比想象中的还要强大的多。顺手把博客里所有代码部分都更新了一下。

完成之后参考Hexo官方更新指南
如果npm install hexo-cli -g这一步出现问题，先做一次hexo clean。还提示permission denied的话可以使用sudo。
第三步npm install hexo --save 一定不要 用sudo，不然之后所有hexo操作就只能sudo才可以执行了。如果不小心手残可以hexo clean砍掉重来。
