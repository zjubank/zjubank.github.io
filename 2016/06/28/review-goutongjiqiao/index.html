沟通技巧复习

发表于 2016-06-28 更新于 2016-07-20 Disqus：

个人观点表达

  1. 简述项目：是什么、做了什么、有什么结果
  2. 告诉听众为什么要听
  3. 直奔主题，提炼精要
  4. 及时根据反馈进行互动

团队有效沟通

如何开会

  1. 事先准备好议程
    1. 一般主持人准备
    2. 提前一天发给与会人员
    3. 会场中必须人手一份
    4. 内容：上次会议纪要的尖端回顾，每个议题的简单介绍，每个议题汇报人和估计讨论时间
  2. 主持人
    1. 按计划推进议程
    2. 控制讨论范围
    3. 引导每个人发言
    4. 控制讨论时间
  3. 推进议程
    1. 每项议程开始汇报人先介绍背景情况
    2. 每个成员都要发表意见，最好有争论
    3. 鼓励从不同角度考虑问题
  4. 会议纪要
    1. 时间、地点、参会人、主持人、纪要人
    2. 对于议程中的每一项，记录
      • 简要讨论内容总结
      • 形成最后的决议
      • 任务落实的责任人、落实时间
    3. 下次会议计划内容、计划时间
    4. 纪要人会后当天整理，群发全体相关人员（包括为与会者）
  5. 总结并计划下次会议内容（主持人）
    1. 对每项议题简要回顾今天的讨论
    2. 一句话陈述讨论做出的决议
    3. 重申责任人和落实时间
    4. 征求大家的意见，计划下次会议

文献检索与有效阅读

  1. 严禁批量下载
  2. 搜索引擎vs数据库（能排序、筛选等）

文献综述

  • 是对某个主题的学术观点和理论方法的整理
  • 是批判性的归纳和评论，不是简单的堆砌
  1. 该领域的研究意义
  2. 研究背景和发展脉络
  3. 目前研究水平、存在问题及可能的原因
  4. 进一步研究的课题、发展方向概述
  5. 自己的见解和感想
  6. 参考文献列表（正文中所有引用必须加标记）

技术文档写作（上）

开题报告

  1. 标题：精准概括内容
  2. 项目背景
    1. 社会发展的需要、课题的意义
    2. 同类产品研究现状
    3. 项目来源及概况
  3. 目标和任务
    1. 目标：最终要做成什么样子，各项指标
    2. 任务：为了达到目标，要先后做哪事
  4. 可行性分析
    1. 技术可行性
    2. 环境条件可行性
  5. 初步方案及关键技术
    1. 怎么做，先做什么后做什么
    2. 关键技术：一般不超过三项
    3. 要文字结合流程图
    4. 技术要“关键”
  6. 预期工作结果
    1. 最终完成什么东西
    2. 满足什么要求
    3. 结果要是名词
  7. 进度计划

毕业设计报告

开篇要有“帽子”，末尾要有“本章小结”
0. 摘要

  1. 300~500字
  2. 社会背景、工作意义
  3. 存在问题
  4. 本文给出了什么解决方案
  5. 通过……（步骤），采用……（技术），做到了……
  6. 经过测试验证，达到了……效果
    1. 项目背景（约3页）
  7. 项目背景及意义
  8. 项目内容
    • 注意不要写成“目标与任务”，是已经完成的
    • 注意点睛：本文作者的主要贡献在
    1. 产品架构概述（不要超过三分之一）
  9. 项目整体需求概述
    • 哪部分是由你解决的
  10. 项目整体系统结构（配图）
    • 哪个模块是你完成的
  11. 关键技术分析
    • 写出实现中用到的、不是你发明的技术
    • 写清楚为什么用这个技术而不是其他
    1. 你的工作
  12. 你负责的模块的需求分析
  13. 你负责的模块的架构设计
  14. 最有技术难度的实现细节
  • 不要贴太多代码
    1. 测试及成果展示
  1. 测试目标（要验证什么指标）
  2. 测试策略（打算怎么测）
  3. 测试结果与分析（重要界面截图）
    • 一定要分析
    1. 总结与展望
  4. 总结全文成果
  5. 展望可以进一步改进的工作
  • 不要重复第一章
  • 不要暴露当前工作的不足

毕业论文

  1. 摘要（同设计报告）
  2. 绪论（3页左右）
    1. 课题背景及意义（类似开题）
    2. 研究现状及本文贡献
      • 是文献综述的缩写
      • 务必点睛：本文作者的主要贡献是……
    3. 本文结构
    • 务必点睛
  3. 基础理论综述（不要超过三分之一）
    1. 标题要换成你所讨论的理论
    2. 描述将要用到的，不是自己提出的理论和算法
    3. 为什么要用这个算法而不是别的
    4. 不是文献综述！
  4. 你的工作
    1. 第2章描述的理论应用于具体课题，解决具体问题的方法
    2. 详细描述算法的推导过程，为什么这样做是对的
    3. 用伪代码和图描述算法流程
  5. 测试与分析（同设计报告）
  6. 总结与展望（同设计报告）

科研论文

  1. Introduction
    1. 本课题如何重要
    2. 前人ABC研究过，提出了什么方案，有什么问题（一句话概括
    3. 我们提出了方法X
    4. X的特点，与ABC的不同
    5. 实验证明X比ABC优越
    6. 本文结构大纲
  2. Related Work
    1. 将前人工作分类
    2. 对前人工作简短描述（几句话）
    3. 与自己的X算法比较

语法修辞

  1. 科学论文必须客观
    1. 尽量使用被动语态
    2. 坚决不用“我”，实在不行用“我们”，“本文作者”
  2. 严谨
    1. 不能用模糊的词语
    2. 不能用“他”，改用“读者”，“人们”
  3. 注意可读性
    1. 禁止长句
    2. 第一次出现必须用全称
    3. 任何代表同一个意义的符号，必须全文统一，大小写、斜正体一致
    4. 程序用等宽字体
    5. 不要拼写数字
  4. 列表
    1. 有序
    2. 无序
  5. 数学相关
    1. 变量名等
  6. 英文论文常用时态

排版惯例

  1. 英文大标题中，单词首字母大写，介词冠词不大写（除非是第一个单词）
  2. 标题上下留空，但不能是第一页的最后一行
  3. 同级标题必须使用同样格式
  4. 标题末尾不出现标点符号
  5. 不到半页不能成为一节（本章小结除外）
  6. 一段最后一行不能出现在下一页
  7. 图标的标题和内容不能在不同页（但是可以和引用在不同页）
  8. 图表必须有含义清晰的标题
  9. 图标题在下，表标题在上
  10. 按章节编号，不能一号到底
  11. 表中数据同一单位的时候，将单位写在标题栏里
  12. 英文逗号后空1格，句号后空2格
  13. 参考文献
    • 按引用顺序编号
    • 按年代编号
